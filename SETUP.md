# Hugo Blog Setup Guide

Complete guide to set up your anonymous Hugo blog on a new machine.

## Prerequisites

- Git installed
- Hugo installed (v0.150.1 or later with extended version)
- GitHub account (0x46society)

## 1. Clone the Repository

```bash
git clone git@github.com:0x46society/0x46society.github.io.git
cd 0x46society.github.io
```

## 2. Set Up SSH Key for 0x46society

Generate a dedicated SSH key for this anonymous identity:

```bash
ssh-keygen -t ed25519 -C "0x46society" -f ~/.ssh/id_ed25519_0x46society
```

Press Enter twice (no passphrase, or add one if you prefer).

Copy the public key:

```bash
cat ~/.ssh/id_ed25519_0x46society.pub | pbcopy
```

Add the key to GitHub:
- Go to: https://github.com/settings/keys
- Click "New SSH key"
- Title: `0x46society-blog` (or your machine name)
- Paste the key
- Click "Add SSH key"

## 3. Configure Git for This Repository

Set the identity for this repo (anonymous):

```bash
git config user.name "0x46society"
git config user.email "0x46society@users.noreply.github.com"
git config core.sshCommand "ssh -i ~/.ssh/id_ed25519_0x46society -F /dev/null"
```

Verify the configuration:

```bash
git config user.name
git config user.email
```

## 4. Initialize Submodules

The theme is a Git submodule, so initialize it:

```bash
git submodule update --init --recursive
```

## 5. Test Locally

Run the Hugo development server:

```bash
hugo server -D
```

Visit: http://localhost:1313

Press `Ctrl+C` to stop the server.

## 6. Writing Posts

Create a new post:

```bash
# Create a new markdown file
touch themes/nostyleplease/content/posts/YYYY-MM-DD-post-title.md
```

Post template:

```markdown
---
title: "Your Post Title"
date: 2025-12-12T21:00:00+08:00
draft: false
---

Your content here...
```

**Important:** 
- Set `draft: false` to publish
- Set `draft: true` to keep as draft (won't show on site)

## 7. Publishing Workflow

1. **Write** your post in `themes/nostyleplease/content/posts/`
2. **Preview** locally with `hugo server -D`
3. **Commit** and push:

```bash
git add .
git commit -m "Add new post: your-title"
git push
```

4. **GitHub Actions** automatically builds and deploys to https://0x46society.github.io/

## Important Notes

### Anonymous Identity
- This repo uses `0x46society` identity
- Email: `0x46society@users.noreply.github.com` (keeps your real email private)
- Dedicated SSH key keeps this separate from other GitHub accounts

### Don't Commit These
The `.gitignore` already handles this, but never commit:
- `public/` folder (auto-generated by GitHub Actions)
- `.hugo_build.lock`
- Personal information

### Local vs Production
- **Local preview:** `hugo server -D` (shows drafts)
- **Production:** GitHub Actions runs `hugo` (no drafts)
- You never need to run `hugo` manually for deployment

## Troubleshooting

### SSH Authentication Failed
Test your SSH connection:

```bash
ssh -T -i ~/.ssh/id_ed25519_0x46society git@github.com
```

Should see: `Hi 0x46society! You've successfully authenticated...`

### Theme Not Loading
Update submodules:

```bash
git submodule update --init --recursive
```

### Site Not Updating
Check GitHub Actions:
- Go to: https://github.com/0x46society/0x46society.github.io/actions
- Verify the workflow completed successfully
- Wait 1-2 minutes after deployment

### Wrong Git Identity
Verify repo config:

```bash
git config user.name
git config user.email
```

Should show `0x46society` and the noreply email.
